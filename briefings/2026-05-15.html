Anthropic 在 SMB 端壓倒性領先，但 Fortune 500 的戰場 Ramp 數據根本看不到

事實背景

Ramp 是一家追蹤 50,000 家以上美國企業公司卡與發票支出的 fintech 平台。其 5 月 AI Index 顯示，Anthropic 在 4 月企業客戶付費採用率達 34.4%，月增 3.8 個百分點，首次超越 OpenAI 的 32.3%（月減 2.9 個百分點）。Anthropic 一年前採用率僅 9%，12 個月內成長 26 個百分點，驅動力是 Claude Code（Anthropic 的 CLI 編程代理工具）。整體企業 AI 採用率同期攀至 50.6%。

So What?

Ramp 的樣本是中小型高成長企業，Fortune 500 大型合約走採購部門年約、不刷公司卡，等於這份數據的最強競爭戰場是看不到的。真正需要交叉驗證的是 Microsoft Azure OpenAI Service 的企業席次，那才是大型合約的代理指標。Anthropic 確實在 SMB 與高成長科技公司這條線壓倒性領先，但用這個數字宣稱「企業端全面勝出」的分析是錯的。

值得追問
  1. Ramp 追蹤的是刷卡支出不是合約續約，Anthropic 的 90 天留存到底長什麼樣，有多少企業只是試用 Claude Code 一個月就回去用 Copilot？
  2. OpenAI 靠 Codex 搶回開發者，Anthropic 如果要守住企業端，最難被 OpenAI 用價格戰或綁定 Azure 通路複製的到底是哪一塊？

Anthropic 的 $100 額度不是福利，是篩選高 LTV 開發者的漏斗

! Agent SDK 從訂閱額度拆出，Anthropic 認可 programmatic 流量沒有人類互動的自然速率上限。
事實背景

Anthropic 通知 Max 5x 訂閱者，從 2026-06-15 起每月可領 $100 額度，專用於 Claude Agent SDK（Anthropic 的程式化 agent 編程介面）與 `claude -p`（Claude Code 的非互動旗標，供腳本使用）等非互動式使用。這筆額度與現有訂閱限額分開，訂閱限額完整保留給 Claude Code、chat 等互動使用。超出 $100 後改走 extra usage 計費，需手動開啟。額度單月不滾存、不可轉讓。

So What?

Anthropic 的真實設計意圖不是補貼現有用戶，$100 對重度 agent 用戶幾小時就燒完。這個額度是要篩出願意開 extra usage 的高 LTV 開發者，同時讓第三方 app 有理由把 Anthropic Agent SDK 當預設選擇而非 OpenAI Assistants API，等於 Anthropic 用這筆錢補貼合作夥伴的用戶試用成本。把 Agent SDK 從訂閱額度拆出來也是正式承認 programmatic 流量沒有人類互動的自然速率上限，未來 Anthropic 的真實營收天花板在 API 計量計費而非固定訂閱。

值得追問
  1. 把 Agent SDK 額度獨立拆出來，Anthropic 目前有多少第三方 app 實際在消耗這條通路，還是生態夥伴數量還遠不足以讓這個設計發揮作用？
  2. Anthropic 的 API 定價要跟 OpenAI 和 Google 打到什麼水位，才能讓開發者不把 agent 流量切去更便宜的模型？

Jensen Huang 缺席訪華團，對賭中國重啟的多頭失去最後一張政治牌

事實背景

Nvidia 市值逼近 $6T，Jensen Huang 個人淨資產超過 $175B，躋身全球前十富豪。但 Trump 本週訪華的企業領袖團名單包含 Tim Cook、Elon Musk 等十餘人，Huang 不在其中。Nvidia 中國市占率自 2025 年 10 月已從 95% 跌至 0%，遭美國出口管制（限制 Nvidia 對中銷售先進 GPU）與中國政府限制雙向夾擊。

So What?

整個 2025 年 Huang 都隨 Trump 出訪用投資承諾換政治讓步，這次唯獨缺席中國行，等於華府明確傳達晶片出口管制的紅線不為 Nvidia 的市值移動。更直接的威脅是 AMD 或 Intel 若在訪問團名單內，競品可能取得中國市場重啟的政治背書窗口，而 Nvidia CUDA 護城河在推論端正被 ONNX Runtime 與華為昇騰工具鏈侵蝕。對賭 Nvidia 中國重啟的多頭現在手上沒有任何政治觸媒可等待。

值得追問
  1. Huang 被排除在訪華團外，但 AMD 和 Intel 有沒有人在團裡，如果有，真正該害怕的不是政策訊號，而是競品拿到中國市場重啟的先手？
  2. Nvidia 中國市占從 95% 歸零後，填補缺口的營收全靠中東和美國超大規模雲，如果 Meta、Google、Microsoft 任一家在 2027 砍 capex 計畫 20%，$6T 的估值還站得住嗎？
TechCrunch / Bloomberg / CNBC AI

Waymo 用 geofencing 繞過了水體偵測問題，不是解決了它

事實背景

Waymo 向 NHTSA（美國國家公路交通安全管理局）申報自願召回（Voluntary recall，廠商主動申報以避免被強制召回）3,791 輛 robotaxi，目標修第五代與第六代自駕系統軟體。導火線是 2026-04-20 一輛無乘客 robotaxi 在 San Antonio 駛入限速 40 mph 道路上的深淹水區並被沖入溪流。Waymo 已推送 OTA update（over-the-air，遠端推送更新）限制極端天候下的營運範圍，San Antonio 服務目前仍暫停，NHTSA 已立案調查。

So What?

Waymo 選擇推送「限制營運範圍」而非「修水體偵測」，這個選擇本身是強信號，暗示問題在感知層，LiDAR 對靜止水面反射不穩定加上視覺難以區分濕路面，是架構層問題而非邏輯錯誤。Geofencing 依賴靜態地圖，在氣候變遷導致淹水位置越來越不可預測的前提下，這不是可擴展的工程方案。更嚴重的風險是城市議會拿這件事拖新城市許可證審批，Cruise 的先例已經證明這條路徑存在。

值得追問
  1. Waymo 最怕的是 NHTSA 調查本身，還是各城市議會拿這件事來拖新城市許可證審批，San Antonio 暫停服務已經是先例，下一個極端天候事件發生在正在審批 Waymo 的城市會怎樣？
  2. OTA 更新選擇限制營運範圍而非真正解決水體偵測問題，這種 geofencing 方式在氣候變遷導致淹水位置越來越不可預測的前提下，是可擴展的工程方案還是技術債的開始？

ZKsync 的銀行隱私解法，把信任從管理員轉移到了電路審計員

事實背景

Matter Labs CEO Alex Gluchowski 接受 60 分鐘訪談，闡述機構區塊鏈邏輯：銀行間互不信任但必須交易，blockchain 解決的是「在不信任系統管理員的前提下強制執行規則」。ZKsync 的 Prividium 是給機構用的隱私保留版本，以 ZK proof（零知識證明，允許驗證計算正確性而不暴露輸入）做「私有執行加公有等級規則強制」雙重保證。Huntington Bancshares、First Horizon、M&T Bank、KeyCorp 四家美國地區銀行正在 ZKsync 上建構代幣化存款（銀行存款的鏈上代表，可在 blockchain 上轉移與結算）網絡，BitGo 提供託管，堆疊預計 2026 年下半年廣泛上線。

So What?

Gluchowski 承認私網失去旁觀者監控市場，但 Prividium 的 ZK 設計讓外部看不到交易細節，這個矛盾在架構上沒有被解決，而是被重新定義：監控責任從市場轉移到 ZK 電路審計員與監管機構。輸家是假設「ZK 等同公有鏈安全級別」的銀行合規官，贏家是能承接電路審計合約的 Formal Verification 公司如 Trail of Bits 和 Certora。四家銀行目前是四個 POC（概念驗證），CoinDesk 用詞是「moving on」和「測試中」，鏈上無公開日均結算量，Matter Labs 的估值是對未來管道定價而非現有營收。

值得追問
  1. 目前鏈上實際結算的日均交易量和金額是多少，四家地區銀行「正在建構」和「已經在結算真實存款」差十萬八千里，還是全部還在沙盒？
  2. 銀行合規規則的模糊性和跨司法管轄區差異，在 ZK 電路裡怎麼處理，誰負責審計這些電路的正確性，這不就是把信任從系統管理員轉移到電路審計員？

Salesforce 不會死，但 agent 寫入資料不產生新座位費，受益的是 Data Cloud

事實背景

a16z 三位作者（Gio Cosenza、Steph Smith、Alex Rampell）發布論述，類比 Facebook 朋友圖譜被 News Feed 取代的歷程：CRM（客戶關係管理系統，企業真理來源資料庫）將從業務員每天打開的中心退化成 agent 的儲物櫃，下個十年的 GTM（Go-to-Market，企業銷售與行銷部門）軟體價值會堆積在 System of Intelligence（SoI，AI 決策與編排層），不在 System of Record（SoR，企業資料記錄層）。a16z 自家 GTM 調查發現，AI 工具普及後 CRM 使用率反而上升。Salesforce 目前市值約 $140B，HubSpot 約 $9B。

So What?

研究指向一個 a16z 自己沒有講清楚的結論：agent 自動寫回 CRM 讓資料量上升，但 Salesforce 的 seat-based 定價是對人類登入次數收費，agent 寫入不產生新座位費，真正受益的是 Salesforce Data Cloud（按數據量計費）與 Agentforce（按 agent 行為收費），不是傳統 CRM 訂閱。這個轉變對 Salesforce 的 ARPU 可能是正的，對 HubSpot 的 seat-count 增長假設是負的。此外，若 Salesforce 自己就是 reasoning layer，a16z 整篇論述描繪的 TAM 就是 Salesforce 的內部功能，獨立 SoI 新創的生存空間比論文說的窄很多。

值得追問
  1. a16z 說 CRM 使用率反而上升，但上升的是 agent 自動寫入的資料量還是業務員主動登入做決策的頻率，如果是前者，Salesforce 的 seat-based 定價模型根本沒受益？
  2. 如果要出貨一個 System of Intelligence 產品，Salesforce 會用 API 限流和資料匯出限制直接掐死通路，a16z portfolio 裡有誰已經被 Salesforce 這樣對待過，怎麼活下來的？
