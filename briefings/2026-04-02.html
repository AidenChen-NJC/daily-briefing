OpenAI 的超級應用是防禦戰，護城河正在被 Google 三面包抄

事實背景

OpenAI 完成史上最大單輪 VC 融資 $122B，估值達 $852B。Amazon 投入 $50B、SoftBank 投入 $30B 共同領投，Nvidia、Andreessen Horowitz、TPG、T. Rowe Price 參與，另有 $3B 來自通過銀行管道的零售投資人。公司目前每月營收 $2B、每週活躍用戶 9 億+，企業營收占比超過 40%。資金用途包括算力基礎設施，以及將 ChatGPT、Codex（AI 程式生成工具）、瀏覽與代理功能合併為單一「超級應用（Superapp，以微信為代表的整合型應用）」。Amazon 的投資合約據報含有 AGI 條款（AGI，Artificial General Intelligence，通用人工智慧，目前無公認定義或觸發標準）——一旦 OpenAI 達到 AGI 門檻，所有合約條款可完全重設。

分析 超級應用策略是防禦動作，防禦對象是 Google，後者正透過 Chrome、Android、Workspace 三條通路同步壓縮 ChatGPT 的入口控制權。

超級應用策略是防禦動作，防禦對象是 Google，後者正透過 Chrome、Android、Workspace 三條通路同步壓縮 ChatGPT 的入口控制權。微信模式的核心是支付閉環，OpenAI 沒有，這個策略只能提高替代成本，無法複製護城河。AGI 條款的可執行性存在根本缺陷：觸發標準由 OpenAI 自己定義，Amazon 買到的是未來談判桌上的席位，不是真正的重議權。

值得追問
  1. 如果 Google 把 Gemini 設為 Chrome 預設助理，OpenAI 的 9 億 WAU 有多少比例是「習慣」而非「黏性」？
  2. AGI 條款由賣方掌握觸發鍵，Amazon 簽下這份合約時，是真的認為這保護了它，還是這只是完成投資的政治包裝？
  3. 超級應用缺少支付層，OpenAI 最終會嘗試進入金融服務嗎，還是它打算用 API 生態作為替代閉環？

Anthropic 洩漏的不是程式碼，是 Claude Code 的整個競爭優勢

⚡ 這次洩漏讓 Claude Code 在 enterprise coding agent 賽道上的差異化敘事，在競爭對手看過之前就提前結束了。
事實背景

Anthropic 因 npm 打包流程的人為錯誤，將 Claude Code CLI 的完整原始碼推送至公開 registry。洩漏內容包含約 1,900 個檔案、50 萬行程式碼、44 個 feature flags（功能標記，控制後端功能開關的代碼變數），以及 3 個未發布專案，含跨對話持久記憶與深度規劃系統。GitHub 鏡像在數小時內達到 4,000+ stars 和 7,000+ forks。這是同週繼 Claude Mythos 模型洩漏後的第二次重大安全事件。Anthropic 聲明屬人為錯誤，無客戶資料外洩。

分析 被公開的不是模型權重，而是 Agent harness（代理框架，包在基礎模型外層、控制工具調用與多步驟規劃的軟體層）的完整邏輯，這才是 Claude Code 的核心競爭力所在。

被公開的不是模型權重，而是 Agent harness（代理框架，包在基礎模型外層、控制工具調用與多步驟規劃的軟體層）的完整邏輯，這才是 Claude Code 的核心競爭力所在。Cursor 或 Windsurf 拿到架構藍圖但沒有對應的模型微調配置，實際效能落差會立刻在 benchmark 上暴露，但 OpenAI Codex 團隊有自己的底層模型可以對應調參，是這次洩漏最直接的受益者。兩週兩次洩漏的複合效應，會在 ISO 27001 和 SOC 2 審查框架下觸發企業採購的供應商風險重新評估，正在和 Anthropic 談判中的金融與醫療垂直整合案是最直接的輸家。

值得追問
  1. 如果 OpenAI Codex 用這份架構藍圖跟著調參，Anthropic 要怎麼在法律上主張競爭傷害，開源授權的邊界在哪裡？
  2. 企業客戶內部安全團隊在引用這兩次事件時，Anthropic 有什麼論述能讓採購決策不轉向 Google Vertex AI？
  3. 兩週內的兩次洩漏是流程問題還是組織問題，這兩種診斷對 Anthropic 的補救動作意味著完全不同的路徑？

Heron Power 的真實客戶不是電力公司，是 Amazon 和 Microsoft

事實背景

美國 70% 的輸電線路和大型功率變壓器已服役超過 25 年，ASCE 2025 年給出 D+ 評級。自 2019 年以來，變壓器需求翻倍但成本漲 80%，美國本土有 30% 的供應缺口且 80% 依賴進口。電費自 2019 年上漲 25% 至 30%，原因不在發電端（再生能源成本持續下降），而在傳輸與配電（占用戶電費約 50%）。Heron Power 由前 Tesla SVP Drew Baglino 創立，以碳化矽（SiC，比傳統矽基功率半導體更耐高壓、效率更高的材料）固態變壓器（SST，Solid-State Transformer）取代傳統油浸鐵芯設備，讓配電節點具備軟體控制與遠端監控能力。公司於 2026 年 2 月完成 $140M Series B，由 a16z 美國活力基金與 Breakthrough Energy Ventures 聯合領投，計畫建設年產 40GW 工廠，目標 2027 年初試產。

分析 Heron 的實際突破口是不受 PUC（公用事業委員會）管制的資料中心業主，Amazon、Microsoft、Google 自建電力基礎設施可以跳過 18 至 36 個月的費率審批流程直接採購。

Heron 的實際突破口是不受 PUC（公用事業委員會）管制的資料中心業主，Amazon、Microsoft、Google 自建電力基礎設施可以跳過 18 至 36 個月的費率審批流程直接採購，受監管的公用事業只是中長期市場。然而這條路徑的集中度風險很高，若任一 hyperscaler 轉向 Siemens 或 ABB 的 SST 方案，Heron 的營收曲線會斷崖式下滑。Heron 的護城河是軟體控制 IP，但在 Wolfspeed（SiC 主要供應商）財務困境的背景下，40GW 產能目標在 2027 年能否兌現本身就是問號。

值得追問
  1. 如果 ABB 或 Siemens 決定認真做 SST，它們的 SiC 替代供應商（STMicro、Infineon）關係讓 Heron 的先發優勢還剩多少個月？
  2. Hyperscaler 直接採購模式讓 Heron 繞開了電力監管，但這也意味著它的定價權完全由三到四家客戶決定，這是護城河還是陷阱？
  3. Wolfspeed 如果繼續財務惡化，Heron 有沒有辦法在不改變核心架構的情況下切換 SiC 供應商？

只有 5% 的人信任 AI 開發者，這個數字會以監管的形式釋放

事實背景

Quinnipiac 大學最新民調顯示，AI 使用率上升 14%，但公眾情緒全面逆行。預期 AI 將縮減工作機會的比例跳升 14 個百分點至 70%。收入差距明顯，年收入 $200K 以上受訪者 52% 認為 AI 利大於弊，年收入 $50K 以下受訪者 60% 認為弊大於利。只有 5% 的受訪者信任 AI 開發者代表自己的利益。74% 認為政府監管不足。最常見使用場景為研究（51%）、寫作（28%）、學校或工作（27%）、數據分析（27%）。

分析 5% 的信任度轉化為實質合規成本，最可能的觸發點是加州的 SB-1047 後續立法，而非聯邦層級。

5% 的信任度轉化為實質合規成本，最可能的觸發點是加州的 SB-1047 後續立法，而非聯邦層級，因為加州可以繞開國會兩黨共識快速落地，且科技公司的運營中心在其管轄範圍內。OpenAI 會是最先被打到的公司，原因是消費者產品規模最大（9 億 WAU）加上本輪 $122B 融資的媒體曝光，讓它成為立法目標的最高優先順序。Google 的遊說能量和 Meta 的法律戰積累讓它們相對有緩衝。

值得追問
  1. 使用率上升同時信任度下降，這表示「不信任但還是在用」正在成為常態，那麼信任赤字還會真的轉化為政治行動嗎？
  2. 如果加州率先立法，OpenAI 有多大意願把核心運營遷出加州，這在實務上可行嗎？
  3. $50K 以下的低收入群體是 AI 最強烈的反對者，也是最可能被取代的勞動力，當這兩件事同時成立，政策設計者的選項清單上有什麼？
