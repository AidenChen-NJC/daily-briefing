SpaceX + xAI IPO：Musk 真正想邊緣化的不是承銷商，是定價權

⚡ 直接散戶配售和現場路演取代傳統銀行主導，這是今日最大的破壞性信號。
事實背景

SpaceX 於 2026 年 2 月完成與 xAI 合併，合併估值 $1.25 trillion，成為史上最大併購案之一。合併後 SpaceX 以「軌道 AI 數據中心」重新定位，計畫 2026 年 6 月在 Nasdaq 上市，目標估值 $1.5T 至 $1.75 trillion，預計融資 $40B 至 $75B 以上，超越 Saudi Aramco 的 $29.4B 成為史上最大 IPO。SpaceX 2025 年營收約 $16B，EBITDA $7.5B，Starlink 月活躍用戶超過 920 萬。計畫中約 30% IPO 股份將直接分配給散戶投資人，並以邀請投資人到德州校區觀看發射取代傳統路演。

分析 SEC Regulation M 要求發行期間的市場穩定由有執照的主承銷商承接…

SEC Regulation M 要求發行期間的市場穩定由有執照的主承銷商承接，FINRA Rule 2310 規定散戶配售需要適當性評估機制，這些法律義務讓承銷商無法被清出局。Musk 實際打掉的是承銷商的定價話語權和路演資訊不對稱優勢，Goldman Sachs 和 Morgan Stanley 最多降級為執行工具。真正的問題在別處：機構投資人在路演第一個問題會是 Starlink 的 ARPU 趨勢和流失率，這兩個數字必須在 S-1 招股說明書中披露，若數字惡化，$1.75T 的估值倍數就缺乏支撐。

值得追問
  1. 如果 S-1 顯示 Starlink 北美用戶在 fiber 覆蓋延伸後流失率上升，IPO 定價會怎麼走？
  2. xAI 的大部分「營收」若實質上是 SpaceX 採購 Grok 的內部轉帳，機構分析師拆解後對整體估值的衝擊有多大？
  3. Musk 讓投資人「飛去看火箭發射再決定投資」這件事，在監管審查下能走多遠？

Meta TRIBE v2：比真實腦掃描更準的模型，最快死掉的是付費腦神經研究公司

事實背景

Meta 於 2026 年 3 月 26 日開源 TRIBE v2（Trimodal Brain Encoder，三模態腦部編碼模型），能預測人類大腦對視覺、聽覺或語言刺激的神經反應。TRIBE v2 訓練資料涵蓋 451.6 小時 fMRI 紀錄、700 名以上受試者，覆蓋 70,000 個腦體素，對比 v1 的 4 名受試者和 1,000 個腦區。模型具備零樣本能力，可對未曾掃描過的受試者進行預測，且合成預測精準度已超越真實 fMRI 紀錄。Meta 以非商業授權開源全部程式碼、模型權重及互動 demo。

分析 Meta 以非商業授權開源、自己保留商業部署權，這個結構說明廣告部門極有可能已在內部測試…

Meta 以非商業授權開源、自己保留商業部署權，這個結構說明廣告部門極有可能已在內部測試神經預測模組用於 Reels 廣告創意優化。最直接的輸家是 Neurons Inc 和 iMotions，它們的商業模式建立在「你必須租我們的掃描設備和分析軟體」，但這個護城河在零邊際成本的合成預測面前結構性崩解，P&G 和 Unilever 的消費者洞察部門不需要 18 個月就能做出切換決定。

值得追問
  1. 如果 Meta 廣告系統已經在用神經預測優化 Reels，用戶同意書裡有任何一行字涵蓋這個用途嗎？
  2. Neurons Inc 和 iMotions 的護城河若只剩個體層級數據，這個窗口期還有多長？
  3. 「超越真實 fMRI」這個基準若只是群體平均，對廣告精準定向的實際價值被高估了多少？

Apple iOS 27：Siri 開放接入第三方 AI，最大受益者是 Apple 自己

事實背景

Apple 計畫在 iOS 27（預計 2026 年 6 月 8 日 WWDC 宣布）透過全新 Extensions 系統開放 Siri，允許 Google Gemini、Anthropic Claude、Microsoft Copilot、xAI Grok、Perplexity 等第三方 AI 接入。用戶可在設定中選擇偏好的 AI 模型處理不同查詢。ChatGPT 自 2024 年 iOS 18 起是唯一外部整合模型，但使用率據報「極低」。Apple 正以 Google Gemini 重建 Siri 底層技術，並計畫對 App Store 中購買的 AI 訂閱抽成。

分析 Apple 控制 API 存取權限是既有架構事實：若 Apple 只向第三方模型傳遞脫敏淺層 context…

Apple 控制 API 存取權限是既有架構事實：若 Apple 只向第三方模型傳遞脫敏淺層 context（聯絡人姓名、日曆事件標題），而把深層 context（健康數據、位置歷史、跨 App 行為）留給自家 pipeline，Claude 和 Grok 在 Siri 內的體驗會被結構性降級為通用問答框。ChatGPT 在 Siri 內「使用率極低」這個先例已經證明：接入 Siri 對 AI 公司的商業價值遠低於直接下載獨立 App，Apple 實際賺到的是讓每家 AI 公司搶著進來的品牌背書，而不是高 ARPU 的訂閱抽成。

值得追問
  1. 各家 AI 公司如果採取「Siri 做曝光漏斗、獨立 App 做付費轉換」的雙軌策略，Apple 能抽到的收入天花板在哪裡？
  2. 如果 Apple 用 Gemini 重建 Siri 底層但同時讓 Google 也在 Extensions 裡和其他 AI 競爭，Google 怎麼看這個安排？
  3. 「使用率極低」的 ChatGPT 整合是 Siri Extensions 生態的預兆還是例外？

Meta 德州 $10B 數據中心：7 倍增幅背後，電力變壓器才是真正卡關的那個點

事實背景

Meta 宣布將德克薩斯州 El Paso 數據中心投資從 $1.5B 提升至超過 $10B，增幅約 7 倍。設施面積從 1.2 百萬平方英尺擴大至 3.1 百萬平方英尺，設計電力容量為 1 吉瓦（GW），預計 2028 年全面上線。高峰建設期約 4,000 名工人在場，完全運營後超過 300 個永久職位。Meta 2026 年全年資本支出預估 $115B 至 $135B，較 2025 年的 $72.2B 大幅躍升。

分析 三個執行瓶頸中，高壓電力變壓器供應鏈是最可能讓時間表滑移的單點風險…

三個執行瓶頸中，高壓電力變壓器供應鏈是最可能讓時間表滑移的單點風險：目前全球交付週期 2 至 3 年，同時受全球所有大型數據中心同步擴張的需求爆炸和製造產能有限的雙重夾擊，無法靠錢加速。NVIDIA H200 和 B200 的 GPU 排程 Meta 可靠預付訂單優先取得，承包商人才問題可靠薪資溢價緩解，但變壓器的物理製造週期是硬性限制，2028 年上線時間表的風險在這裡，不在其他地方。

值得追問
  1. 如果 Meta 沒有提前在 2024 年以前下訂變壓器，2028 年上線的承諾是否已經在滑移？
  2. 從 $1.5B 到 $10B 若主要是 inference 而非訓練的成本爆炸，對 GPU-as-a-service 公司的長期商業邏輯意味著什麼？
  3. 所有大型科技公司同步自建算力的情況下，電網供應商和變壓器製造商成為隱形瓶頸，誰會最先提出系統性解法？

Shield AI $1.5B 融資：Anduril 的反制窗口只有一次，而且不在今天

事實背景

Shield AI 於 2026 年 3 月完成 $1.5B 融資，估值達 $12.7B，較 2025 年 3 月的 $5.3B 增長 140%。融資催化劑是美國空軍選定 Shield AI 的 Hivemind 自主飛行軟體，在 Anduril 的 YFQ-44A「Fury」無人機上運行，作為 CCA（Collaborative Combat Aircraft，協作作戰飛機）計畫的 AI 層。Hivemind 設計可在 GPS 和通訊完全中斷下獨立操作。競爭對手 Anduril 同期競逐 $8B 融資，目標估值 $60B。

分析 Shield AI 在 Fury 上的護城河不是合約，而是認證週期…

Shield AI 在 Fury 上的護城河不是合約，而是認證週期：DoD 軟體在作戰平台上的替換需要重新通過 DO-178C 航空軟體認證和實際飛行測試，這個流程以年計。Anduril 踢走 Hivemind 的唯一合理窗口是在下一代平台設計時直接拒絕 Hivemind 接口，這也是為什麼 Shield AI 需要用這 $1.5B 快速把 Hivemind 擴展到更多機型，讓自己成為跨平台標準，讓替換成本高到 Anduril 不理性。

值得追問
  1. 如果 Hivemind 的多機協同在實戰中是預編程戰術劇本而非真正即時分散式決策，對手一旦識別劇本觸發條件，Shield AI 的估值假設會怎麼重新定價？
  2. Anduril 全棧整合的策略在硬體和軟體都自己控制的下一代平台上有多大優勢？
  3. 軍用 AI「軟體平台標準」和「硬體生態整合」這兩種架構，在 10 年時間軸上哪個更容易形成壟斷？

Claude Code Dispatch & Computer Use：從工具進化成「數位員工」，但企業安全會把它擋在門外

事實背景

Anthropic 於 2026 年 3 月 28 日宣布 Claude Code 四項新功能：Dispatch（從手機啟動本機電腦上的開發工作）、Remote Control（透過網頁或手機 App 接管終端機工作階段）、Channels（透過 iMessage、Telegram、Discord 等通訊平台對 Claude Code 下達指令，透過 MCP 伺服器連接）、/loop（設定定期執行任務，如每日 PR review 或夜間測試套件）。Dispatch 和 Computer Use 適用於 Pro 和 Max 方案，Channels 適用於所有付費方案，均為研究預覽版，Computer Use 限 macOS。

分析 在有 MDM 和 SOC2 合規要求的企業環境中，Claude Code 作為擁有 shell 存取權的常駐代理…

在有 MDM 和 SOC2 合規要求的企業環境中，Claude Code 作為擁有 shell 存取權和螢幕控制能力的常駐代理，落在「高權限非人類身份」威脅模型分類，類似 PAM 的管控目標。Anthropic 目前的審計日誌和權限隔離能力遠未達到 CyberArk 或 BeyondTrust 等企業安全工具的標準，這意味著 Claude Code 的企業滲透在金融、醫療、政府等強 IT 治理組織會被安全團隊直接擋住，短期市場只有 security-light 的科技新創。Channels 功能才是對 Cursor 的真正威脅，不是 /loop，因為它讓 AI coding assistant 的觸達點從 IDE 延伸到任何通訊介面，繞過了 Cursor「IDE 內嵌入」的分發優勢。

值得追問
  1. 如果 Anthropic 不推出符合 SOC2 Type II 和 PAM 標準的企業版，Claude Code 的天花板是否就只有個人開發者和初創公司？
  2. Channels 讓用戶透過 iMessage 控制 Claude Code，這對 Cursor 的核心用戶黏性造成的侵蝕速度會有多快？
  3. /loop 把 Claude Code 從工具變成「一直在跑的同事」，但在沒有審計日誌的情況下，誰來對它的決策負責？

Pharos RWA L1：前螞蟻集團團隊的公鏈賭注，比其他 RWA 項目多一樣東西

事實背景

Pharos 是由前螞蟻集團區塊鏈基礎設施團隊創建的模組化 Layer 1 區塊鏈，專為 RWA（Real World Assets，真實世界資產上鏈）基礎設施設計。技術亮點包括：DTVM（Deterministic Transaction Virtual Machine，同時支援 EVM 和 WASM 智能合約的統一執行環境）、SPN（Special Processing Network，應用專屬執行層，繼承主網安全但獨立處理高需求工作負載），目標主網 30,000 TPS。同一研究中，restaking 協議 Kelp 推出 KUSD 穩定幣，以真實貿易應收帳款支撐，Kelp 現有 TVL $1.25B，年化收入 $300 萬，KUSD 目標 10% 以上年化收益，來源是借款人還款而非代幣激勵，目標市場包含全球每年超過 $25 trillion 的貿易應收帳款。

分析 Pharos 創辦人的螞蟻集團背景讓它天然具備與傳統金融機構對話的能力…

Pharos 創辦人的螞蟻集團背景讓它天然具備與傳統金融機構對話的能力，但它選擇在完全去中心化的公鏈上構建，這個取捨讓它比單純在公鏈上做 RWA 的項目多了一樣東西：說服傳統金融機構上鏈的語言和關係，而不只是技術方案。研究未對 Pharos 競爭對手位置（相對於 Ethereum L2、Solana、Sui 在 RWA 賽道的份額）提供直接比較數據，這個結論的信心屬中等。

值得追問
  1. 前螞蟻集團團隊做公鏈，螞蟻集團本身（或阿里巴巴生態）是否可能成為第一批機構級採用者，還是反而形成競爭？
  2. KUSD 的 10% 年化收益若來源是借款人還款，這個借款人的信用風險如何被定價，誰來承擔違約損失？
  3. 30,000 TPS 的目標在沒有足夠應用和用戶的情況下，技術指標本身能吸引多少機構資金？
