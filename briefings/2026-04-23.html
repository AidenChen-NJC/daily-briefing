SpaceX 花 $60B 買的不是 Cursor，是 IPO 後的廉價 scrip

事實背景

Cursor 是月活近千萬的 AI 程式碼編輯器（IDE），2026 年底預估 ARR $6B+。4/21 SpaceX 向 Cursor 股東發出正式 offer：於 2026 夏季 SpaceX IPO 完成後的 12 個月內，SpaceX 有權以 $60B 全額收購 Cursor；若最終不行使，直接支付 $10B 現金作為「合作工作」補償。這等同一份 call option（買入選擇權），執行價 $60B、權利金 $10B。此公告直接終止了 a16z 共同主導、Thrive Capital 跟投、Nvidia 加碼的 $2B 融資輪（pre-money 估值 $48B）。Cursor CEO Michael Truell 當日接受此安排，公司維持獨立運營至行權日。

So What?

a16z 和 Thrive 的「十億美元級 windfall」有兩層前提同時成立才能兌現：SpaceX 行使買斷權，且 IPO 後公開市場估值足夠接近 Musk 自訂的 $1.25T。若 SpaceX IPO 後市值落在 $600-800B 區間，scrip（以股票支付的對價）縮水 30-40%，LP 實拿的現金等值遠低於 headline 數字，補貼的是追高 SpaceX IPO 的散戶。Musk 的真正策略是把 IPO 當作取得「可用於併購的流動性籌碼」的工具，而非募資。

Amazon 的 $33B 不是賭 Anthropic，是把自己逼上 Trainium 這條路

事實背景

4/20 Amazon 宣布加碼投資 Anthropic：$5B 即期注資，視商業里程碑再追加最多 $20B，加上先前 $8B，Amazon 對 Anthropic 的最大曝險達 $33B。作為對等承諾，Anthropic 同意未來 10 年對 AWS 投入超過 $100B，採購涵蓋 Trainium2、Trainium3，以及尚未發佈的 Trainium4 晶片，鎖定訓練與推論容量達 5GW（十億瓦）。Anthropic 目前 ARR（年度重複性收入）已達 $30B，是 2025 年底 $9B 的三倍以上，主要驅動來自 Claude 在 agentic coding（AI 自動寫程式的應用場景）中的底層使用。

So What?

這份協議的核心不對稱性在於：若 Trainium3/4 未能達標，Amazon 承受的是 $25B 投資的未實現虧損，Anthropic 只是承受效率損失，而且沒有公開的合約退出條款。真正被鎖死的是 Amazon，這份 IOU 的實際用途是逼 Amazon 內部讓 Trainium 成功，而不是逼 Anthropic 留在 Trainium 上。研究指向的明確輸家是 Nvidia，失去 Anthropic 的增量推論訂單；明確贏家是 Marvell 和台積電，整個 Trainium 供應鏈被強制 lock-in。

  2. $30B ARR 裡有多少是 Amazon 自家採購換算過來的，這個「左手右手」的比例，決定了 Anthropic 到底有多值錢？

Google 的 A2A 協議真正的目的是控制企業 agent 的入口

事實背景

4/22 Google Cloud Next 2026，Sundar Pichai 公布四塊完整堆疊。硬體層：TPU 8 世代首次把訓練與推論拆分為 TPU 8t 和 TPU 8i 兩顆獨立 ASIC。平台層：Gemini Enterprise Agent Platform，包含 Agent Designer、Inbox、Long-Running Agents 等模組。協議層：A2A（Agent-to-Agent Protocol，規範 agent 和 agent 之間如何溝通的協議）正式推出，定位為企業 multi-agent 協作標準。安全層：Agentic Defense 整合 Google Threat Intelligence 與 Wiz（以色列雲端資安公司，2025 年 Google 以 $32B 收購）。同時宣布 $750M 合作夥伴基金，注入 120,000+ 個合作夥伴生態。

So What?

A2A 和 Anthropic 的 MCP（Model Context Protocol，規範 agent 和工具之間溝通的協議）並非同層競爭，MCP 管 agent 和工具的介面，A2A 管 agent 和 agent 的協調。Google 強推 A2A 的真實目的是控制 agent identity registry（agent 身份登記系統），也就是企業多 agent 系統的入口。Google 的結構性優勢不是 TPU，而是 Workspace 已在全球 30 億用戶裝置上，若 A2A 讓 Workspace agent 成為企業 agent 網路的預設協調者，MCP 生態就被降格為工具接口層。真正的輸家是 LangChain 和 CrewAI 等 framework，它們的 orchestration 邏輯將被夾在 A2A 和 MCP 兩個大廠協議之間邊緣化。

值得追問
  1. 企業的 CIO 在選「agent 協調層」時，30 億人已在用的 Gmail/Docs 有多重的分量？
  2. 如果 A2A 成為企業標準，Anthropic 的 MCP 生態最終的地位會是「工具插槽」而非「協調者」，這改變了什麼？
  3. $750M 平均分給 120,000 個 partner 只有 $6,250，這筆錢能買什麼？

0G × Qwen 的「可驗證」AI，驗不了 Alibaba 到底有沒有審查你的 prompt

事實背景

4/21 0G Foundation（鏈上 AI 基礎設施）與 Alibaba Cloud 宣布合作，讓 Qwen 3.6 Plus（阿里雲最新 LLM）成為首個「on-chain 可取用」的主流語言模型。開發者原本需透過 Alibaba Cloud API key 付費，現在改為在 0G 上以 token（鏈上代幣）購買推論配額；推論本身仍在 Alibaba Cloud 執行，0G 提供 verifiable compute（可驗證計算）層，確認結果未被竄改。0G Labs CEO Michael Heinrich 表示：「inference runs on Qwen, and verification runs on 0G。」

So What?

0G 的 verifiable compute 只能驗證「output hash 與 Alibaba 回傳的一致」，無法驗證「Alibaba 跑的是未審查版 Qwen 而非合規過濾版本」。在中國網路審查框架下，Alibaba Cloud 有法律義務對特定 prompt 進行內容過濾，這意味著 0G 的信任假設從根本上是斷裂的，它驗證的是一致性而非完整性。贏家是 Alibaba，獲得「去中心化」包裝卻不承擔實質約束；輸家是相信 0G 能提供抗審查 AI 輸出的 onchain agent 開發者。

值得追問
  1. 一個「可驗證但可能已被過濾」的 LLM，對需要抗審查的 use case 有任何價值嗎？
  2. Alibaba 選擇 onchain payment rail 的動機如果真的是繞過出口管制，當 OFAC 第一次 enforcement action 來時，整個 stack 會剩下什麼？
  3. 如果 agent 帶錢包即可付費、不需 KYC，誰是實際的風控承擔者？

BTC $78K 的漲法，宣告了它已不是加密原生資產

事實背景

4/22 Trump 宣布將 Iran 停火框架延長至下一個談判窗口，S&P 500 與 Nasdaq 同步上揚。BTC 當日從 $76,341 開盤、美東午後突破 $78,000（11 週新高），日漲 +4.3%、週漲 +6.6%。過去 7 天 $427M 空單（借券賣出後被迫平倉的部位，稱為 short squeeze）被清出。Polymarket（以 USDC 結算的事件合約平台）上「BTC ≥$80K by end-April」合約的 YES 機率在 24 小時內從 36% 升至 76%。ETH 同步上漲至 $2,327（+0.5%）。這次 rally 的觸發來源是地緣政治緩和，而非 ETF 資金流入、減半敘事或任何加密原生事件。

NY AG 起訴 Coinbase 的關鍵，是 Dodd-Frank 裡一個「gaming 排除條款」

事實背景

4/22 紐約州總檢察長 Letitia James 對 Coinbase 和 Gemini 提起訴訟，指控兩家的 prediction markets（事件合約市場）違反紐約州 Penal Law 的賭博規定。具體指控：未取得 NY State Gaming Commission 牌照、允許 18-20 歲使用（紐約 mobile sports betting 要求 21+）、未繳賭場同級稅。Coinbase CLO Paul Grewal 在 X 回應：「Prediction markets are federally regulated national exchanges」，表明防禦主軸是 CFTC（美國商品期貨交易委員會）的聯邦管轄權優先於州 gambling 法。此次訴訟未列 Polymarket 為被告。

So What?

Coinbase 的聯邦 preemption 防禦有一個致命弱點：Dodd-Frank §5c(c)(5)(C) 明確把「gaming 相關」的事件合約排除在 CFTC 管轄之外。若 NY AG 成功論證 Coinbase 的體育和娛樂類合約屬於此排除條款，CFTC 的保護傘對這部分產品直接失效，而 Coinbase 目前仍無正式 DCM（指定合約市場）牌照明確覆蓋 prediction market。這場訴訟的勝負，取決於 Coinbase 合約標的中體育加娛樂類的佔比，比例越高，聯邦防禦越脆弱。

值得追問

  2. 如果 NY 勝訴，Kalshi 緊接著面對 50 個州各自主張管轄，整個美國 prediction market 市場會變成什麼樣？

AI 電力缺口讓 $380M 流進了船廠核反應爐，資本已分岔為兩條路

事實背景

4/21 Blue Energy 完成 $380M growth round，由 VXI Capital 領投，Engine Ventures、At One Ventures、Tamarack Global 跟投。商業模式是在船廠預製模組化核反應爐（SMR，小型模組化反應爐，單機組低於 300MW），整體海運至現場安裝，交付週期 48 個月 turnkey（交鑰匙工程，供應商全包設計、建造到調試），是業界標準 8-12 年的三分之一至六分之一。設計總容量 1.5GW（十億瓦），首個專案落地德克薩斯，2026 Q3 動工，目標客戶為 AI data center。前 US NRC（美國核管理委員會）主席 Greg Jaczko 擔任顧問。

So What?

這筆融資最重要的信號不是「核能復興」，而是時程。Google Cloud Next 同日公開 5GW+ 電力需求、Anthropic-Amazon deal 鎖定 5GW，兩者合計已超過美國 2025 年全年新增基載電力容量的規劃數字，主電網擴張速度確實無法滿足 2026-2028 AI 需求的一半。Blue Energy 的 48 個月 turnkey 是 hyperscaler 願意付溢價的核心原因，不是技術創新，是速度。資本已在 2026 正式分岔為「模型層 VC」和「能源層 VC」兩個生態，兩條路線的盡職調查邏輯和人才圈子完全不同。

Coinbase 的量子報告把 Bitcoin 最大的 FUD 變成了可量化的 6.9M 個錢包

事實背景

4/15 Coinbase 量子運算顧問委員會（由 Stanford 密碼學家 Dan Boneh 主持，成員含 Ethereum Foundation researcher Justin Drake 和 EigenLayer 創辦人 Sreeram Kannan）發表 50 頁 position paper。核心結論：現時 blockchain 仍安全，但 fault-tolerant quantum computer（容錯量子電腦，能執行任意長演算法的量子機器）的到來愈來愈 plausible，遷移必須從現在開始。Bitcoin 有 6.9M 個錢包已暴露公鑰，量子電腦成熟後為優先攻擊目標；post-quantum signature（能抗量子電腦的新簽章演算法）大小是當前 10-100 倍，替換可能讓 block size 膨脹至 38 倍。Algorand 和 Aptos 被評為「最就緒」的主流鏈，Bitcoin 和 Ethereum 舊版簽章（ECDSA）為最脆弱。

So What?

這 6.9M 個錢包包含大量「pay-to-public-key」格式的早期礦工地址（含 Satoshi 時代），公鑰已永久暴露在鏈上，不需要任何操作即可被未來量子機器反推私鑰，且大量地址早已無主（持有人失去私鑰或已逝），根本無法主動遷移。這份報告是 Algorand 和 Aptos 迄今最具份量的 institutional narrative，贏家是他們，輸家是 Bitcoin 的「不可竄改性」信仰，一旦社群辯論「要不要 freeze 無法遷移的舊錢包」，Bitcoin 將面臨 2017 年 block size 之爭之後最大的政治共識分裂壓力。

值得追問
  1. 如果 Satoshi 的幣因為公鑰暴露而「可被量子攻擊」，Bitcoin 社群要怎麼決定要不要凍結它們？
  2. Algorand 和 Aptos 獲得「最就緒」這個 institutional narrative，能不能轉化成實際的機構配置資金流入？
  3. 量子時程被預估在 2030-2040 年，在這之前的 5-15 年，這個議題對 BTC 的市場定價有多少實際影響？
