Anthropic 帳面收入反超 OpenAI，但股東自購讓真實 ARR 可能只有帳面的六成

事實背景

Anthropic 正與投資人洽談以 $900B 估值完成新一輪融資，term sheet 尚未簽署但預計兩週內落定，將首次正式超越 OpenAI 的 $850B 估值。Anthropic 宣布年化營收 ARR（Annualized Revenue Run Rate，最近月份營收乘以 12）已達 $30B，內部 run rate 接近 $40B，對比 16 個月前的 $9B，成長約 3.3 倍。Counterpoint Research 數據顯示 Anthropic 在 Q1 2026 以 31.4% 的 LLM（大型語言模型）收入市占率超越 OpenAI，但月活用戶僅 134M，OpenAI 約為 900M。資金來源：Google 承諾最高加碼 $40B，Amazon 已投 $5B、達里程碑後再投最高 $20B。

So What?

Anthropic 的 $40B run rate 高度依賴 Google Cloud 和 Amazon Bedrock 的轉售通路，這兩家股東合計承諾投資上限達 $65B，且雲端合約通常包含使用量承諾。若股東自購消費佔 ARR 的 30% 至 50%，真正第三方 ARR 可能僅 $20B 至 $28B，對應 $900B 估值的 PS 倍數將從表面的約 22.5x 跳升至 32x 至 45x，已超出任何可參照的 SaaS 先例。這個數字需要 Anthropic 的招股書才能確認，目前是區間估算，不是實測值。

值得追問
  1. Anthropic 最難被抄走的到底是模型本身，還是它嵌進 AWS 和 GCP 採購流程的通路鎖定，哪一層才是真正的護城河？
  2. 如果這筆 $900B 融資兩週內落定，Google 和 Amazon 投錢給 Anthropic 讓估值上漲，再把持股增值算進自己的 AI 利潤，Alphabet 的股東到底是在買 Google 的 AI 能力，還是在買一個對 Anthropic 的槓桿 pass-through？

OpenAI 的真正問題不是 ChatGPT 慢下來，是 900M 免費用戶正在拖垮它的成本結構

事實背景

OpenAI 在 2025 年底同時錯過內部營收與用戶成長目標。CFO Sarah Friar 表示，若營收增速無法跟上，OpenAI 將難以履行為衝刺規模而簽訂的長期算力合約，這類合約通常採 take-or-pay 結構（不論是否實際使用，均須按約定量付款）。All-In Podcast 指出，OpenAI 在企業與開發者市場同時被 Anthropic 蠶食，程式碼生成工具 Codex 面臨 Anthropic Claude Code 的正面競爭，同時消費端受到 Google Gemini 壓制。

So What?

OpenAI 的核心矛盾在於，它用 900M 免費用戶撐起的算力基礎建設，正在被 Anthropic 以 134M 付費用戶更高效地變現。放棄免費用戶會失去規模敘事，但維持免費用戶則讓每個付費用戶要補貼的算力成本更高。Codex 的真正競爭對手不是 Google，是 Anthropic Claude Code 和 Cursor，兩者都不需要從 900M 免費用戶的包袱裡討生活。

值得追問
  1. OpenAI 的 take-or-pay 算力合約總義務規模未公開，這筆表外負債在 $850B 估值裡被定價了嗎？
  2. 900M 月活創造的營收竟低於 Anthropic 的 134M 月活，SoftBank 和 Tiger Global 在 $850B 那輪建模時，單位經濟的假設到底是什麼？
Fortune AI

Google 和 Amazon 的「AI 利潤」有一半是對同一資產反覆估值，不是真實現金流

事實背景

Fortune 4/30 報導指出，Google 與 Amazon 最新一季財報中所謂的 AI 業務利潤暴增，經拆解後約有一半來自帳上 Anthropic 持股的估值上修，即 mark-to-market（按市價計量，未實現的估值變動影響當期帳面利潤）。Google 對 Anthropic 承諾最高 $40B，Amazon 已投入 $25B。隨著 Anthropic 估值走向 $900B，兩家公司下季財報的帳面 AI 利潤將進一步被推升。

So What?

若 Anthropic 下一輪估值持平或 down round，Google 與 Amazon 在 GAAP 下需反轉先前認列的未實現收益，這不是「利潤少賺」，而是可能從正轉負的一次性衝擊。這個結構和 Cisco 在 2000 年的 vendor financing 有一個關鍵差異：Cisco 是借貸（有違約觸發損失），Google 和 Amazon 持有的是股權，估值可以「持平」而非「下跌」，讓迴圈維持更久，但一旦 Anthropic 下輪 down round，衝擊是一次性且非線性的。明確的輸家是相信 Google 和 Amazon「AI 利潤」真實性的被動型 ETF 持有人。

值得追問
  1. 如果 Anthropic 下一輪 down round 或估值持平，Google 和 Amazon 各自的 AI 業務利潤會從正轉負嗎？華爾街分析師的模型裡有沒有把這個 mark-to-market 風險單獨拆出來？
  2. 拿掉 Anthropic 持股 mark-to-market 後，Google Cloud AI 和 AWS Bedrock 各自的 AI workload 毛利率與客戶淨留存到底是正還是負？

Anthropic 被 Pentagon 排除在外，真正受益的是 Palantir，不是 OpenAI

事實背景

Pentagon 與 7 家大型科技公司簽訂 AI 合作協議，唯獨將 Anthropic 排除在外。Anthropic 的 Constitutional AI（一種讓模型按「憲法」規則自我約束輸出的訓練方法）在武器化、監控等場景的拒答率遠高於同業。這對在 LLM 收入市占率第一的 Anthropic 而言，意味著聯邦防務市場這個未來年規模可能達數百億美元的客戶群，暫時對其關閉。

So What?

簽約七家中，最能在實際軍事場景替代 Claude 的不是 OpenAI（消費導向），而是 Palantir（已有 ATO 授權、資料管線與現場部署經驗）和 Microsoft（Azure Government 雲搭配 OpenAI 模型）。Anthropic 不進場讓 Palantir 的護城河直接加深，因為 Palantir 的競爭力從來不是模型本身，而是「拿到授權、接好資料、留住安全官」的整合能力，Anthropic 缺席讓這個能力更難被模型廠商繞過。這個結論的信心是高的，Palantir 的 ATO 與 DoD 整合能力是公開事實。

值得追問
  1. Anthropic 不進場反而讓誰的護城河加深了？如果我今天要做 defense AI，最難複製的是 Palantir 和 Anduril 已拿到的 ATO 和資料管線，還是模型能力本身？
  2. 七家簽約公司裡哪一家的模型能力最接近 Claude，能在實際軍事場景中成為 Anthropic 的功能替代，而非只是合規替代？

o1 在回顧性病歷上跑贏醫師，但這個成績幾乎不可能進入真實急診室

事實背景

TechCrunch 5/3 報導 Harvard 對 LLM 在醫療場景的研究結果。在真實急診室案例中，OpenAI o1 模型在診斷準確度上略勝或持平於兩位人類主治醫師（attending physician），且差距在 ER triage（急診初次分流）階段最明顯。研究強調 LLM 的優勢在於資訊整合速度與不知疲倦。

So What?

這項研究是在回顧性病歷文本上跑的，輸入的是乾淨結構化資料，而非真實 ER 工作流中的多模態噪音，包括即時體徵數字、護理師口頭觀察與影像。歷史上，在回顧性資料上達到「勝過醫師」的 AI 系統，進入真實 EHR（電子病歷系統）整合部署的轉換率趨近於零。真正的 GTM 瓶頸不是模型準確度，而是 Epic 和 Cerner 的整合授權費與醫院法務的責任歸屬條款，OpenAI 目前沒有自己去談這些的跡象，定價權在整合商手中。

值得追問
  1. 「略勝兩位主治醫師」是在回顧性病歷上跑的還是真正嵌入 ER 工作流即時分診的？前者是 benchmark，後者才是牽引力，兩者轉換率歷史上趨近於零。
  2. 如果我明天要賣 AI 分診產品給醫院，真正的 GTM 瓶頸是 Epic 和 Cerner 的 EHR 整合權限，還是醫院法務的責任歸屬問題？OpenAI 自己會去談這些還是只當模型供應商？

UiPath 說七成 AI 試點失敗是因為流程爛，但它自己的 agentic 功能數字從未公開

事實背景

The Rundown 5/3 專訪 UiPath CMO Michael Atalla。核心數據：70% 至 80% 的 agentic AI（具自主行動能力、能在工作流中執行多步任務的 AI 系統）試點無法進入正式部署，主因為 agent 工具各自孤立、無法銜接到業務目標。入門級開發者職缺自 2024 年起下滑約 20%，同期資深職缺成長。Atalla 提出：「AI makes good workflows faster and bad ones more expensive」。

CLARITY Act 把「善意活動」的定義權留給 SEC，Coinbase 說支持，但這是一場賭注

事實背景

參議院銀行委員會的 Tillis 與 Alsobrooks 5/1 公布 CLARITY Act 對穩定幣收益的妥協文本：禁止穩定幣業者提供「功能上或經濟上等同銀行存款」的收益，但允許「bona fide activities」（真實商業活動）產生的獎勵。Coinbase 與 Circle 5/2 立刻表態支持並施壓銀行委員會推動 markup（法案正式審議程序）。

So What?

這個結構有一個技術上的死角：SEC 要靠鏈上數據區分「真實商業活動獎勵」與「變相利息」，在技術上存在根本障礙。PoS（權益證明）節點的 MEV 收入、質押獎勵與流動性挖礦獎勵在鏈上的呈現方式幾乎相同，現有鏈上分析工具（Chainalysis、Nansen）無法從交易結構本身區分「意圖」。這意味法規的實際執行將依賴發行商的自我申報，監管套利的空間不會消失，只是把合規責任轉移到 Circle 與 Coinbase 身上。

值得追問
  1. 「bona fide activities」的定義權最終落在 SEC 還是 OCC？Coinbase 和 Circle 急著背書，是因為已拿到私下保證，還是在賭解釋權會對自己有利？
  2. 如果 staking yield 被包裝成 bona fide reward，PoS 驗證節點的 MEV 收入算不算「等同存款利息」？這條線在協議層根本劃得清嗎？

Meta 不自建、接 Stripe：穩定幣結算的預設路徑正在被一家公司壟斷

事實背景

穩定幣總市值本週達 $321.7B，創歷史新高，最近一週（4/26 至 5/3）淨流入 $1.08B。同週企業動作：Visa 在全球穩定幣結算試點再加 5 條鏈、總計支援 9 條；Meta 開始向部分創作者提供穩定幣支付，技術由 Stripe 支援，這是 Meta 自 Diem 倒台後首次重返加密支付；Walmart 投資的 OnePay 也加入穩定幣支付。

So What?

值得追問

  2. 如果我是競爭者要打 Stripe 的穩定幣 rail 位置，最難打穿的是它和 Meta、OpenAI 的整合關係，還是底層多鏈結算的合規授權？護城河在哪一層？
